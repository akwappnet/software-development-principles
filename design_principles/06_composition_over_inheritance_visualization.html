‚ùå Inheritance Approach

Rigid family tree - everyone must fit into predetermined categories

ü¶∏‚Äç‚ôÄÔ∏è Superhero Inheritance Hierarchy
class Animal {
    eat() { return "eating"; }
}

class Bird extends Animal {
    fly() { return "flying"; }
    eat() { return "eating seeds"; }
}

class Fish extends Animal {
    swim() { return "swimming"; }
    eat() { return "eating plankton"; }
}

// Problem: What about a duck?
// Ducks can fly AND swim!
// Where does Duck go in this tree?

class Duck extends Bird {
    swim() { return "swimming"; } // Awkward!
}

// Even worse: What about a penguin?
// Penguins are birds but can't fly!

    fly() { 
        throw new Error("Penguins can't fly!"); 
    }
}
Problems with this approach:
‚Ä¢ Rigid structure - hard to add new combinations
‚Ä¢ Some animals break the rules (penguin "birds" that can't fly)
‚Ä¢ Diamond problem - multiple inheritance conflicts
‚Ä¢ Deep hierarchies become hard to understand
‚Ä¢ Changing parent class breaks all children

‚úÖ Composition Approach

Flexible building blocks - mix and match abilities as needed

üß© Ability-Based Composition
// Create reusable abilities
const flyingAbility = {
    fly() { return "soaring through the sky"; }
};

const swimmingAbility = {
    swim() { return "gliding through water"; }
};

const eatingAbility = {
    eat(food) { return `eating ${food}`; }
};

// Compose animals by combining abilities
function createAnimal(name, abilities) {
    return {
        name,
        ...abilities
    };
}

// Now we can create any combination!
const duck = createAnimal("Duck", {
    ...flyingAbility,
    ...swimmingAbility,
    ...eatingAbility
});

const penguin = createAnimal("Penguin", {
    ...swimmingAbility,
    ...eatingAbility
    // No flying - perfectly fine!
});

const eagle = createAnimal("Eagle", {
    ...flyingAbility,
    ...eatingAbility
});
Benefits of this approach:
‚Ä¢ Flexible - any combination of abilities
‚Ä¢ Easy to add new abilities without changing existing code
‚Ä¢ No forced inheritance - only include what you need
‚Ä¢ Reusable components across different animal types
‚Ä¢ Changes to one ability don't affect others
