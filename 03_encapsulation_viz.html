✅ With Encapsulation
public class BankAccount {
  private double balance;
  private String accountNumber;
  private String pin;

  public void deposit(double amount) {
    if (amount > 0) balance += amount;
  }

  public double getBalance() {
    return balance;
  }
}

🔒 Data Protection

Internal data is hidden and protected from unauthorized access. Like keeping valuables in a safe.

🎛️ Simple Interface

Complex internal operations are hidden behind simple, easy-to-use methods. Like a TV remote.

🛡️ Validation Control

All data access goes through controlled methods that can validate and sanitize inputs.

🔧 Easy Maintenance

Internal implementation can change without affecting external code that uses the class.

🐛 Reduced Bugs

Controlled access prevents accidental corruption of internal state and data.

📖 Better Understanding

Clear public interface makes it obvious how the class should be used properly.

🔒 What is Encapsulation?

Encapsulation is like putting your valuables in a safe - you hide the internal details and provide controlled access through a simple interface. It's one of the fundamental principles of object-oriented programming that bundles data and methods together while hiding the internal implementation.

Think of it like:

  • 🏦 Bank Vault: Your money is protected, but you can access it through ATM (controlled interface)
  • 📺 TV Remote: You don't need to understand electronics, just press buttons
  • 🚗 Car: You don't control pistons directly, just use steering wheel and pedals

The Golden Rule: Hide what can be hidden, expose only what needs to be exposed.
